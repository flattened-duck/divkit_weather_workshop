# Структура презентации v2 — «BDUI и DivKit» (под воркшоп-приложение)

> Это **план структуры** (скелет), не финальные слайды. На слайдах со скринами/кодом
> пока стоят плейсхолдеры `[СКРИН: …]`, `[КОД: …]`, `[СКРИНКАСТ: …]` — наполним из
> готовых `app/` и `backend/`.

## Принципы

1. **Показали → разобрали.** Сначала живое приложение и вау-эффект (темы/язык на лету),
   потом по частям «как это устроено».
2. **Проприетарка — в общих словах.** Flex / Tovarisch / AppHost не показываем как наш
   стек; упоминаем «в проде Яндекса есть внутренний слой над открытым DivKit, идеи те же».
   Весь предметный код — из нашего открытого `app/` + `backend/`.
3. **Каждая подтема = часть приложения.** Несколько экранов · персистентность · языки · темы.
4. **Аудитория** — Android-разработчики: аналогии (Variables/StoredValues ≈ DataStore,
   выражения ≈ derivedState, и т.п.).
5. Лейауты — из нашего шаблона (`talk/deck.py`): `title_plain`, `quote` (разделители),
   `blank` (скрин/диаграмма), `compare`, `two_blocks`, `header_text`, `numbered`,
   `big_thesis`, `agenda`, `questions`.
6. **Схемы в приоритете.** Код — короткими сниппетами (≤ ~6–12 строк) только там, где
   важно увидеть синтаксис; всё остальное объясняем диаграммами.
