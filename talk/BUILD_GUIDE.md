## Рабочий цикл (ОБЯЗАТЕЛЬНО итерируйся визуально)
1. Пиши build-скрипт `build_deck.py`, который `import deck`, собирает слайды по плану, `prs.save(".../bdui_divkit.pptx")`.
2. Рендерь: `.venv/bin/python render.py .../bdui_divkit.pptx .../render 110`
3. **СМОТРИ на PNG-и** через Read. Проверяй: нет ли обрезанного текста, переполнения, наезжающих блоков, пустых серых плейсхолдеров.
4. Правь координаты/размеры/тексты → пере-рендерь → смотри снова. 2-3 итерации до чистоты.

## Бренд (из темы шаблона)
- Цвета (hex): ink `222A3A` (текст), ink2 `575E6E` (вторичный), red `F8604A` (главный акцент), yellow `FFC812`, фоны: lilac_bg `F5F4FD`, pink_bg `FFF4F2`, yellow_bg `FFF9E6`, grey `CFCDDD`, white `FFFFFF`.
- Шрифты: `YS Text Medium` (заголовки), `YS Text Regular` (текст) — установлены, рендерятся.
- Формат: 16:9, холст 67.74 × 38.1 см. Левое поле ~3.8 см, заголовок сверху ~2.45 см.

## API `deck.py`
```
prs = deck.new_deck()                      # шаблон без 184 демо-слайдов
s = deck.add(prs, "<layout>")              # добавить слайд по friendly-имени
deck.set_title(s, "текст")                 # заполнить TITLE плейсхолдер
deck.fill_bodies(s, ["t1", ["a","b"], ...])# заполнить BODY плейсхолдеры по порядку (стр или список=абзацы)
deck.notes(s, "тезисы спикера")            # заметки докладчика (для лекции)

# free-form (на blank-слайдах для диаграмм), координаты в СМ:
deck.textbox(s, x,y,w,h, text, size=18, color="ink", bold=False, align="left", anchor="top")
deck.card(s, x,y,w,h, fill="lilac_bg")                       # скруглённая плашка
deck.card_with_text(s, x,y,w,h, title, body, fill=, accent=) # плашка с текстом
deck.chip(s, x,y, "тег", fill="red", color="white")          # пилюля-тег
deck.arrow(s, x1,y1, x2,y2, color="ink2")                    # стрелка
deck.arrow_line(s, x1,y1,x2,y2, head=True/False)
deck.number_badge(s, x,y, n)                                 # кружок с цифрой
```
Цвета принимай по имени из палитры (`"red"`, `"ink"`, ...) или hex-строкой.

## Каталог friendly-лейаутов (когда что брать)
| friendly | назначение |
|---|---|
| `title_plain` | ТИТУЛ — **бери этот** (без серой картинки). set_title + fill_bodies([подзаголовок, роль]) |
| `agenda` | оглавление/содержание (нумерованные пункты) |
| `quote` | секционный разделитель / ключевая мысль (крупный текст красным) |
| `big_thesis` | большой тезис на слайд |
| `header_text` | заголовок + один текстовый блок |
| `title_subtitle_text` | заголовок + подзаголовок + текст |
| `three_blocks` | 3 колонки текста |
| `two_blocks` | 2 текстовых блока |
| `compare` | сравнение (бэкенд vs клиент, было/стало) |
| `stages` | этапы/шаги потока (дозапросы, версионирование) |
| `numbered` | нумерованный список |
| `cards` | карточки |
| `blank` | ЧИСТЫЙ ХОЛСТ под кастомные диаграммы (cards + arrows) |

## Готчи (найдено на смоук-тесте — НЕ повторяй)
1. **Титул:** используй `title_plain`, а не `title` — у `title` есть picture-плейсхолдер, который без картинки рендерится серым прямоугольником.
2. **Узкие плейсхолдеры титула:** подзаголовок/роль держи КОРОТКИМИ (≤ ~35 символов на строку), иначе текст обрезается справа.
3. **Диаграммы:** делай на `blank`, заголовок отдельным `textbox` (x=3.8, y=2.45, size=32-36, bold). Карточки не делай слишком высокими — высота под контент (часто 6-8 см хватает), вертикальный центр текста при необходимости через `anchor="middle"`.
4. **Стрелки между карточками:** y = вертикальный центр карточки; x от правого края левой до левого края правой.
5. **Длина текста на слайде:** это слайды, не документ — 2-5 коротких пунктов, не абзацы. Подробности уходят в `deck.notes()` (заметки спикера).
6. **Размер файла:** ~30 МБ (бренд-медиа шаблона тянется в файл) — это нормально, не пугайся.

## Заметки спикера = лекция
Тезисы-подсказки лекции встраиваются в заметки каждого слайда через `deck.notes(s, text)`.
Их пишет отдельный агент (lecture) — координацию делает оркестратор.
